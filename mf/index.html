Manual Configuration (If You Don’t Use the Link)

1. Set the modem preset

  1. Open the Meshtastic app or Web UI for your node.
  2. Go to Radio / Modem settings.
  3. Set the modem preset to:
    MediumFast
  4. Save / sync the settings to the device.

2. Configure the primary channel

  1. Open the Channels section for your node.
  2. Edit the Primary channel (or create one and set its role to Primary).
  3. Set the channel name to:
    OmaMesh
  4. Set the channel key to:
    QrgAqFFTvRST/F0hISEmqA==
  5. Save / sync the channel settings to the device.

If your device already has a different primary channel for other networks, you may want a dedicated “test node” so that your normal mesh traffic isn’t changed.

How to Verify You’re on the Right Network

  • After changing settings, reboot or power-cycle your node so the radio uses the new modem preset and channel.
  • In the Meshtastic app, check the node list for other devices on OmaMesh – you should start seeing traffic when others are nearby.
  • Send a short text (for example: OmaMesh test) and confirm that others on this test network can see it.

Scope & Etiquette

  • This is a test / proof-of-concept network on the MediumFast preset, not a general-purpose or emergency channel.
  • Keep messages short and relevant to testing connectivity, range, and node behavior.
  • If you also participate in other local Meshtastic networks, double-check which node and channel you’re using before sending important traffic.

Troubleshooting Tips

  • No traffic seen? Confirm:
    • Modem preset is exactly MediumFast.
    • Channel name is OmaMesh (spelling matters only for your UI, not RF, but it helps avoid confusion).
    • Channel key is QrgAqFFTvRST/F0hISEmqA== with no extra spaces.
  • Still not working? Try re-applying the config using the link at the top of this page, then power-cycle the node.
