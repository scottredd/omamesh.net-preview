Two approachable ways to join the local mesh.

OmaMesh is a simple guide for people in the Omaha metro who want to get on the air and understand the two main paths used locally: Meshtastic and MeshCore.

Both are meant to support decentralized mesh networking. This page gives the short version and points you to NebraskaMesh.net for the current shared NebraskaMesh MeshCore network.

Choose a path

Meshtastic

Meshtastic is a good fit if you want simple out-of-the-box use, easy phone pairing, and a large beginner-friendly ecosystem.

It is great for ad-hoc local use, though its messaging model is more limited than MeshCore's public and private messaging focus.

Visit Meshtastic.org

MeshCore

MeshCore is designed around public and private messaging and is the main platform behind the shared NebraskaMesh effort.

The current NebraskaMesh focus is the default high-speed SF7 standard, while some local users still value the older SF9 approach for longer-range links.

Visit MeshCore.io

NebraskaMesh.net

Use NebraskaMesh.net for current shared-network settings, map, events, and coordination for the NebraskaMesh MeshCore network.

While mesh networking is decentralized by design, NebraskaMesh uses some coordination to help build the strongest practical mesh possible across Nebraska's challenging terrain and nearby surrounding states.

What to do next

Use this site as a local on-ramp, then branch out.

If you want quick ad-hoc setup, Meshtastic is an easy place to start. If you want the Nebraska shared MeshCore network, use NebraskaMesh.net for the latest settings and map.

OmaMesh previously advocated legacy SF9 MeshCore settings because they helped connect longer-range nodes when the local mesh was thinner. Today, the shared Nebraska network is centered on default SF7, while legacy SF9 remains part of the local history and may still appeal to some operators.

Settings

Two MeshCore setups you may still hear about locally.

OmaMesh historically pointed people to an SF9 configuration for longer-range reliability. NebraskaMesh is moving to the default SF7 standard for its shared network, with the change announced for May 31, 2026.

NebraskaMesh standard (SF7)

This is the default USA/Canada MeshCore preset and the stated NebraskaMesh direction for the shared network.

  • Preset: USA / Canada (Recommended)
  • Frequency: 910.525 MHz
  • Bandwidth: 62.5 kHz
  • Spread Factor: SF7
  • Coding Rate: CR5 default

Notes: SF7 is faster and is the default narrow-band MeshCore setting. NebraskaMesh.net has announced a network-wide switch to SF7 on May 31, 2026, along with updated repeater firmware and a move toward 2-byte repeater IDs for the shared network.

Legacy OmaMesh MeshCore (SF9)

Based on the old OmaMesh guidance for users who wanted the local long-link settings that were commonly recommended earlier on.

  • Preset: USA / Canada (Recommended)
  • Frequency: 910.525 MHz
  • Bandwidth: 62.5 kHz
  • Spread Factor: SF9
  • Coding Rate: use default CR5 for normal nodes
  • Coding Rate for repeaters: CR8

Notes: SF9 trades speed for better weak-signal and longer-range performance. Old OmaMesh guidance specifically said repeater operators should change Coding Rate to 8 and reboot.

Check NebraskaMesh.net before programming a radio for the shared network, since live network standards can change faster than this page.

Future efforts

Possible ways to support both standard and long-range use.

One possible future direction is asking legacy SF9 operators to transition to a different frequency so it can continue serving longer-range needs without conflicting with the standard MeshCore defaults.

Another is custom long-range bridge hardware that could help different spread factor configurations interoperate more cleanly, especially for rural operators trying to connect distant nodes.
